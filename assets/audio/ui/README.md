# UI Audio (reserved)

Reserved for a future **Interactive Stings** layer — short, one-shot sounds
tied to real interactions (placing a sticker, publishing, a page turn) — per
`docs/ATMOSPHERE_ENGINE.md`'s long-term vision.

## Currently empty, and out of scope for V1

Interactive Stings are explicitly **not** part of Atmosphere V1 (MLAS)'s
responsibility list — `js/audioManager.js` only ever handles the Foundation
bed and the optional World ambience overlay. See
`docs/ATMOSPHERE_V1_BLUEPRINT.md` §12 ("Future Extension Points") for how
this folder is expected to be used once that work is scoped and approved.

This folder exists now, empty, purely so the top-level `assets/audio/`
structure is fully laid out per the V1 blueprint's own folder plan — not
because anything reads from it yet.
